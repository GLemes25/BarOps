A action deleteEvent em actions/event-actions.ts está falhando silenciosamente por conta de restrições de Foreign Key. Como o Drizzle com neon-http não suporta transações, precisamos excluir os dados filhos manualmente antes de excluir o pai.

Edite a função deleteEvent(id: number) para executar as exclusões na seguinte ordem sequencial:

await db.delete(eventDrinks).where(eq(eventDrinks.eventId, id));

await db.delete(eventLabor).where(eq(eventLabor.eventId, id));

await db.delete(eventMaterials).where(eq(eventMaterials.eventId, id));

E por fim: await db.delete(events).where(eq(events.id, id));

Certifique-se de que os imports das tabelas (eventDrinks, eventLabor, eventMaterials) e da função eq estejam corretos no topo do arquivo. Retorne o objeto de sucesso ou capture o erro no catch (mantendo o revalidatePath). Não crie comentários."

Um toque rápido no seu Frontend
Para você não ficar no escuro quando essas coisas acontecerem de novo, recomendo muito alterar o seu handleDelete no arquivo events-table.tsx para algo assim:

```
const handleDelete = async (id: number) => {
    const result = await deleteEvent(id);
    if (!result.success) {
      console.error("Erro ao deletar:", result.error);
      alert("Não foi possível excluir o evento.");
    } else {
      router.refresh();
    }
  };
```
