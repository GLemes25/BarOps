"No mobile, a minha AppSidebar não tem como ser aberta pois falta o botão de gatilho. Vamos resolver isso aplicando as melhores práticas de UI/UX com um Header global fixo.

Execute os seguintes passos rigorosamente:

Criação do Topbar:

Crie o componente components/top-header.tsx.

Importe SidebarTrigger de @/components/ui/sidebar.

O componente deve retornar um <header> com as seguintes classes do Tailwind: sticky top-0 z-40 flex h-14 w-full items-center gap-4 border-b bg-background/80 px-4 backdrop-blur sm:px-6.

Dentro deste header, adicione o <SidebarTrigger className="-ml-1"/>.

Integração no Layout Base (app/layout.tsx):

Importe o novo TopHeader.

Modifique a estrutura dentro do <main> para que o cabeçalho fique sempre no topo e o conteúdo role (scroll) abaixo dele.

A nova estrutura dentro de <body> deve ficar exatamente assim:

TypeScript
<Providers>
<SidebarProvider>
<AppSidebar />

<div className="flex flex-1 flex-col overflow-hidden">
<TopHeader />
<main className="flex-1 overflow-y-auto bg-muted/20">
{children}
</main>
</div>
</SidebarProvider>
</Providers>
Ajuste de Estilo (Opcional mas recomendado):

O fundo bg-muted/20 no <main> ajuda a destacar os Cards brancos do dashboard. Certifique-se de que o visual fique premium.

Não crie comentários no código e garanta que os imports estejam corretos."

O que essa solução entrega:
Sticky Header: Quando você rolar a lista de eventos ou a tela inicial, o menu vai ficar grudado no topo.

Backdrop Blur: O conteúdo vai passar "borrado" por baixo do cabeçalho, o mesmo efeito que a Apple usa no iOS.

Acessibilidade: O SidebarTrigger nativo do shadcn já vem configurado com acessibilidade para leitores de tela e atalhos de teclado (como Cmd+B para abrir/fechar).
