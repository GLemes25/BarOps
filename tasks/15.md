"O usuário apontou uma regra de negócios essencial: a mão de obra possui um custo base por um período de horas, e um custo por hora extra. Além disso, precisamos garantir que existam as tabelas de 'Catálogo' e as tabelas de 'Alocação no Evento'.

Execute os passos abaixo seguindo as .claude/rules:

Reestruturação do Schema (db/schema.ts):

Atualize a tabela labor_catalog. Ela deve conter: id (serial), role (text), baseCost (decimal 10,2), baseHours (integer), e extraHourCost (decimal 10,2).

Atualize a tabela material_catalog. Ela deve conter: id (serial), name (text), defaultCost (decimal 10,2).

Recrie as tabelas pivô para os eventos (se não existirem):

event_labor: id, eventId (fk events), laborCatalogId (fk labor_catalog), quantity (integer).

event_materials: id, eventId (fk events), materialCatalogId (fk material_catalog), quantity (integer).

Faça o push das alterações para o banco (drizzle-kit push).

Refatoração do CRUD Base de Mão de Obra (actions/labor-actions.ts e app/labor):

Atualize as actions para refletirem os novos campos do labor_catalog (baseCost, baseHours, extraHourCost). Certifique-se de manter o revalidatePath('/labor').

Em labor-form.tsx, atualize o Zod schema e adicione os 3 campos numéricos (Custo Base, Horas Base, Custo Hora Extra). Use rótulos descritivos.

Em labor-table.tsx, exiba as novas regras na tabela. Exemplo de formatação de coluna: 'R$ 180,00 (por 5h) + R$ 60,00/h extra'.

Refatoração do CRUD Base de Materiais (actions/material-actions.ts e app/materials):

Garanta que as actions, formulário e tabela do Catálogo de Materiais estejam refletindo apenas name e defaultCost. O material será cadastrado como um valor unitário padrão de locação (ex: 'Balcão' - R$ 150,00).

Limpeza:

Se o arquivo de seed (db/seed.ts) existir, atualize-o para refletir a nova estrutura de labor_catalog para evitar erros na próxima vez que for rodado.

Não crie comentários no código e utilize o shadcn para todos os novos inputs."
